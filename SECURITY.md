# Security Policy

## Reporting a security issue

Please do not post security issues, private prompt stores, credentials, API keys, conversation exports, or sensitive logs in a public GitHub issue.

Use GitHub's private security-advisory reporting for this repository when available. If private reporting is unavailable, contact the repository owner through the GitHub profile and include only the minimum information needed to establish impact and reproduce the issue safely.

## Scope

MiniPromptLib is local-first. Reports concerning corrupt-store handling, unintended data disclosure, unsafe shell behavior, or an adapter exceeding its bounded-context contract are in scope. Do not send live personal prompt libraries as proof of an issue.
