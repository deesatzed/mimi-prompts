## What This Project Does

MiniPromptLib is an offline-first terminal navigator for AI-assisted work. Given bounded current context, it detects a workflow moment — uncertainty, failure, checkpoint, completion, and more — and offers at most three reusable mini-prompts, weighted by workflow relevance first and bounded `selection_count` frequency second; only an explicit numbered choice increments `selection_count`.

The installed `mini` command owns a local JSON prompt store and interaction loop, plus a full lifecycle around it: view-before-select, undo-on-back, folder organization, a consent-gated text-harvesting pipeline (`mini harvest`), and a usage-feedback loop (`mini feedback`/`mini stats`). Host integrations are thin, project-local templates using the same JSON contract. The app does not read full conversation history, does not call a model for the core workflow, and does not auto-save, deploy, or alter global host settings.

**Tech Stack:** Python 3.11+, standard library only (zero runtime dependencies), setuptools, `unittest`, dependency-free HTML/CSS/JavaScript for the static landing page.
**Architecture Pattern:** Local CLI plus a reusable Python engine, with opt-in project-local adapter templates. No server, database, account, telemetry, mandatory model, or network service.
