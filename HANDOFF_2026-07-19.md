## What This Project Does

MiniPromptLib is an offline-first terminal navigator for AI-assisted work. Given bounded current context, it detects a workflow moment—uncertainty, failure, checkpoint, completion, and more—and offers at most three reusable mini-prompts. It balances workflow relevance with bounded `selection_count` frequency without presenting a huge menu; only an explicit numbered choice increments `selection_count`.

The installed `mini` command owns a local JSON prompt store and interaction loop. Host integrations are thin, project-local templates using the same JSON contract. The app does not read full conversation history, does not call a model for the core workflow, and does not auto-save, deploy, or alter global host settings.

**Tech Stack:** Python 3.11+ (verified locally with 3.13.9), standard-library runtime, setuptools, `unittest`, dependency-free HTML/CSS/JavaScript.  
**Architecture Pattern:** Local CLI and reusable Python engine with opt-in adapter templates; no server, database, account, telemetry, mandatory model, or network service.
