## Artifact-P0-AlienActivation — First-principles reframing

### Root human need

During an AI-assisted build, a person needs a short next-step prompt that fits the *current workflow moment* without opening, reading, or trusting a giant prompt catalog. The critical job is orientation and momentum: “the agent just asked a question,” “the build failed,” “I am uncertain,” or “a section just completed.”

The implementation addresses that job with a bounded `ContextPacket` (`last_agent_message`, `last_user_message`, `build_status`, optional explicit state) [models.py:L20-L26], a deterministic classifier [classifier.py:L8-L34], and a three-item page function [ranking.py:L73-L77].

### Five alien questions

| Question | Finding | Design consequence |
| --- | --- | --- |
| If no prompt library existed, what would a user actually ask for? | “Give me the next useful way to think, not a menu.” | Workflow state is the first gate, not semantic search alone. |
| What does the user pay for when a suggestion is wrong? | Attention and confidence; the user must re-explain context. | Show 2–3 choices, provide `more` and retry rather than a long list. |
| Is repetition evidence of usefulness or habit? | Both. Frequency is useful only inside a logically relevant workflow cohort. | Filter matching workflow states before applying bounded frequency. |
| What must never happen? | The tool must not scrape private conversations, silently save language, or mutate global agent configuration. | Bounded supplied context, preview-first capture, repo-local templates. |
| What is the smallest organism that creates value? | A local CLI that seeds, suggests, accepts a number, and remembers that explicit choice. | No hosted service, account, LLM, or web app in v1. |

**Alien conclusion:** MiniPromptLib is not principally a prompt database. It is a local *workflow-transition navigator* whose stored prompts are reusable decision aids.

## Artifact-P1-OrganismContract — Product contract

### Organism declaration

> Workflow Prompt Navigator helps a person recover the right next reasoning move during an AI conversation or software build. Given only the current bounded workflow context, it returns no more than three explainable, reusable mini-prompts; it learns only from explicit selections; and it keeps ownership, storage, and integrations local.

### Quadruple contracts

| Contract | Commitments | Evidence or measurement |
| --- | --- | --- |
| Value | Return a useful small set for question, checkpoint, completion, failure, proposal, undecided, capture, or general context. | Eight-state model [models.py:L9-L17]; ten deterministic tabletop scenarios in `docs/reports/tabletop-results.json`. |
| UX | The primary path is `seed → suggest → choose number`; recovery is `more` or revised context; capture previews before save. | CLI action vocabulary [cli.py:L168-L208]; capture preview/confirmation [cli.py:L211-L227]. |
| Technical | Offline, standard-library runtime; human-readable schema-v2 JSON; atomic writes; legacy reads; installed `mini` console. | No runtime dependencies [pyproject.toml:L5-L13]; atomic persistence [storage.py:L44-L60]; release check [scripts/release_check.py:L26-L86]. |
| Integrity | Never inflate ranking through display; never silently replace corrupt storage; never auto-install integrations or scrape conversations. | Explicit selection mutation [navigator.py:L38-L47]; corruption error [storage.py:L19-L41]; integration boundary [docs/integrations.md:L3-L11]. |

### Success metric ledger

| Metric | v1 threshold | Current status |
| --- | --- | --- |
| Choice-set size | At most 3 per page | **Verified** by `page_ranked_prompts` and CLI tests. |
| Logical guardrail | An irrelevant frequent prompt cannot beat a clear workflow match | **Verified** by ranking test and state candidate gate [ranking.py:L40-L70]. |

## Artifact-P2-ArchitectureDecisionPack — Divergence before convergence

### Candidate architectures

Scores are 0–5; higher is better. They are decision scores, not implementation measurements.

| Architecture | Feasibility | Novelty | Resilience | Evolvability | Simplicity | Total | Decision |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | --- |
| A. Local deterministic core + CLI + thin adapters | 5 | 3 | 5 | 5 | 5 | **23** | **Selected** |
| B. Native agent hooks as the primary runtime | 3 | 4 | 3 | 3 | 3 | 16 | Defer: tight coupling and global configuration risk. |
| C. Hosted web/mobile prompt manager with account sync | 2 | 2 | 2 | 3 | 1 | 10 | Reject for v1: creates auth, privacy, sync, and deployment burdens. |
| D. Append-only workflow event ledger with generated prompts | 3 | 4 | 4 | 4 | 3 | 18 | Research direction: improves auditability but adds data-model and consent complexity. |

### Selected architecture

The selected architecture keeps a reusable domain core, a workflow layer, a terminal interface, and non-installing adapter references:

```
bounded context -> deterministic state -> state-gated ranking + bounded frequency
      |                                              |
      +----------------------------------------------+-> page of <= 3 -> explicit select
                                                               |                  |
                                                           more/retry        atomic local JSON
```

The classifier gives explicit adapter state precedence [classifier.py:L8-L12]. The ranker permits frequency only after the workflow-state candidate gate [ranking.py:L40-L64]. The navigator owns selection mutation and progressive navigation [navigator.py:L19-L67]. This split is appropriate because it makes the most consequential behavior inspectable and testable without model calls.

### ADR ledger

| ADR | Decision | Why now | Revisit trigger |
| --- | --- | --- | --- |
| ADR-1 | JSON v2 remains the local store | Single-user, inspectable, portable, no infrastructure. | Library grows to thousands of prompts or concurrency is needed. |
| ADR-2 | Workflow state gates frequency | Prevents learned habit from replacing logical relevance. | Evaluation shows state labels are too coarse. |
| ADR-3 | Capture is preview-first | User wording is an asset, not raw telemetry. | Structured editing needs emerge from observed use. |
| ADR-4 | Integrations stay thin and opt-in | Avoids scraping, duplicate ranking logic, and global config mutation. | A target agent proves a stable, permissioned hook contract. |
| ADR-5 | No mandatory LLM | Core value must work when offline and be repeatable. | A separate, explicitly opt-in quality experiment beats deterministic baseline. |

### Complexity budget

Budget: 100 points maximum; no component should exceed 15. Current planned/implemented allocation is 73.

| Component | Points | Why |
| --- | ---: | --- |
| Schema and safe storage | 9 | Version envelope, legacy read, atomic write, corruption stop. |
| Models and classifier | 10 | Small explicit vocabulary with inspectable precedence. |
| Ranker and pagination | 12 | State gate, token overlap, bounded frequency, stable ordering. |
| Navigation and capture | 14 | Session state, nesting, retry, preview/confirmation. |
| CLI and JSON protocol | 15 | Human loop plus scriptable adapter surface. |
| Seed conversion | 8 | Protected authored Markdown → deterministic metadata. |
| Package, docs, and reference adapters | 5 | Installable command and non-installing examples. |

**Budget verdict:** Within budget. The 15-point CLI is the highest-pressure component; avoid adding web, sync, or hook-specific business logic to it.

### Dependency tribunal

| Dependency or surface | Verdict | Rationale |
| --- | --- | --- |
| Python standard library | Acquitted | Runtime behavior is self-contained. |
| `setuptools>=68` | Probationary build dependency | Needed only for packaging; pin is modest but should receive routine compatibility checks. |
| Optional Ollama client | Probationary/isolated | Legacy mining path visibly exits when unavailable [cli.py:L79-L101]; it is not part of core proof. |

### SOTA benchmark, used as market context—not a parity claim

| Existing product capability | What it demonstrates | Gap/opportunity for this app |
| --- | --- | --- |
| [OpenAI Codex skills](https://developers.openai.com/codex/skills/) provide reusable task instructions and workflows. | Prompt/workflow assets can be project-local, reusable, and tool-aware. | MiniPromptLib should remain a chooser and memory layer, not duplicate a host’s skill runtime. |
| [Claude Code skills](https://code.claude.com/docs/en/slash-commands) package reusable instructions that can be invoked or loaded when relevant. | Progressive loading and workflow packaging reduce repeated instruction. | The unique testable hypothesis here is context-state-sensitive *small-set retrieval* across hosts. |
| [Cursor Rules](https://docs.cursor.com/context/rules-for-ai) provide scoped, version-controlled persistent agent instructions. | Persistent instructions are a mature category. | Rules are mostly static context; MiniPromptLib aims to decide which small prompt fits a changing build moment. |

No controlled comparison, user study, adoption data, or benchmark against those products exists. Therefore **do not claim SOTA performance or market superiority**.

## Artifact-P4-VibeAutopsy — Brutal quality review

### What feels real

- The product respects the central UX promise: it makes a small recommendation rather than dumping a catalog.
- The state gate is simple enough to explain, and the reason text exposes both state and selection count [ranking.py:L60-L66].
- Capture protects user language by making saving a second, explicit step [capture.py:L25-L57].

### What feels prototype-like

| Issue | Severity | Why it matters | Remediation decision |
| --- | --- | --- | --- |
| First use needs a separate `seed` command; an empty store provides no guided bootstrap. | High | New users can encounter “No matching prompts” before the first value moment. | **P0 release blocker for public launch.** Add a zero-data first-run guide or explicit seed suggestion. |
| “Nested” interaction is not a full composition UX in the terminal; it asks whether to add another prompt but does not expose a clear compose/preview/confirm sequence in the interactive loop. | High | The stated nesting value is hard to discover and potentially misleading. | **P0 release blocker.** Make preview, compose, and back visible actions. |
| Root `core.py` plus `minipromptlib/core.py` compatibility indirection is transitional layering. | Medium | It increases cognitive load and risks drift. | Keep for v1 compatibility; establish a deprecation plan before v2. |
| Frequency is safe in unit tests but has no user-facing explanation of how it changes over time or reset path. | Medium | A personal system needs legible control over learned habit. | P1: surface counts and a reversible reset/export workflow. |
| There is no actual installed Codex/Claude integration, only templates. | Medium | The promise of in-conversation usefulness requires a manual setup step. | Correctly documented; keep opt-in. Validate one host integration in a consented pilot. |
| The legacy LLM mining surface remains in the same CLI. | Low | It can distract from the deterministic product story. | Keep compatible but separate it clearly in help/docs. |

**Vibe verdict:** coherent, careful, and technically credible for an internal/pilot CLI; not yet polished enough to call a public-facing “frictionless workflow navigator.”

## Artifact-P5-AdversarialValidation — Attack the decision

| Attack | Evidence | Result | Required response |
| --- | --- | --- | --- |
| “Frequency will make the favorite prompt dominate.” | Matching workflow candidates are selected before frequency, and frequency is capped at a 2-point normalized boost [ranking.py:L40-L70]. | Mitigated in unit coverage. | Add real-use inspection and reset capability before broad rollout. |
| “An unsafe write can destroy a personal prompt library.” | Parse failure raises `StorageCorruptionError`; write uses same-directory temp, `fsync`, and `os.replace` [storage.py:L19-L60]. | Mitigated for tested cases. | Add backup/export affordance and crash/fuzz tests in v1.1. |
| “The tool will silently scrape sensitive chat.” | Only supplied bounded fields exist in `ContextPacket`; docs prohibit scraping and global config changes [models.py:L20-L26], [docs/integrations.md:L3-L11]. | Mitigated by design. | Preserve this boundary in any future native adapter. |
| “The classifier will misread nuanced context.” | It is intentionally keyword-deterministic with a general fallback [classifier.py:L18-L34]. | Accepted limitation. | Pilot with misclassification review, then consider user-selected state shortcut. |
| “A shell CLI is inaccessible or too slow under pressure.” | Keyboard path is compact but terminal width, discoverability, and assistive workflow were not evaluated. | Unresolved. | UX P0 before public launch. |
| “Passing tabletop tests equals product-market proof.” | Tests are deterministic fixtures; no users or outcome data were collected. | Rejected claim. | Treat release evidence as engineering proof only. |

## Artifact-P6-FutureProofMap — Progressive disclosure roadmap

### P0 — before public or broad pilot release

1. Guided first-run state: empty library explains the one command that seeds the authored panel, without mutating anything automatically.
2. Complete interactive nesting: explicit `preview`, `compose`, and `back` actions with no hidden selection side effect.
3. A short terminal onboarding test with 3–5 target users or representative operators.

### P1 — after the first observed sessions

1. User-visible selection history/reset/export control.
2. Misclassification feedback that records a correction locally only with explicit confirmation.
3. A single explicitly installed host adapter, tested against the versioned JSON contract.
4. Backup/recovery guidance and property/fuzz tests for persistence.

### P2 — only when evidence supports it

1. Session-scoped workflow event ledger, subject to an explicit privacy/retention design.
2. Optional local semantic retrieval experiment compared against the deterministic baseline.
3. Multi-device sync only after durable consent, conflict, recovery, and threat-model work.

## Artifact-P7-GovernanceGuardrails — Boundaries and stop rules

| Guardrail | Operational rule |
| --- | --- |
| Privacy | Accept only context explicitly passed to the tool; do not scrape conversations or persist raw transcripts by default. |
| Autonomy | A displayed suggestion is never an action. Save, compose, install, publish, push, or global configuration changes require separate explicit confirmation. |
| Learning | Increment `selection_count` only on a chosen number; do not infer preference from display, scrolling, preview, or cancellation. |
| Data safety | Fail closed on unreadable JSON and preserve source bytes; use atomic writes. |
| Scope | No accounts, hosted service, sync, analytics pipeline, mandatory model, or standalone GUI without a new approved goal. |
| Evidence | Label fixture/tabletop evidence as engineering validation, never as user adoption or outcome improvement. |

The existing build is a valid engineering milestone with fresh, reproducible proof. The P0 UX items above are the narrowest path to a credible public pilot. Do not add hosted infrastructure, model dependence, or a GUI to compensate for these basic interaction gaps.
